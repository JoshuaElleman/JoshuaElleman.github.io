# Enhancement Two

## Artifact Overview
Artifact two is an application that uses vectors and structs in C++ to create complex data structures that can be sorted and searched algorithmically. It was used to organize and sort college courses and store their prerequisites. I wrote this application for CS 300 in April of 2024. 

## Justification for Inclusion in ePortfolio
I selected this application for inclusion in my ePortfolio because it demonstrates my ability to implement algorithms and data structures in a practical way. Several components of this artifact showcase my skills in algorithms and data structures such as the user of vectors and structs. The use of vectors and structs demonstrates my ability to work with dynamic data structures to store and manage data. The implementation of a merge sort illustrates my understanding of efficient sorting algorithms which can scale up better than other solutions such as the original selection sort algorithm. The search algorithm also shows my ability to design and implement algorithms for data retrieval. The artifact was significantly improved through the enhancement process. Upon reviewing the original code, I found that the search function contained flawed logic and did not correctly locate courses or output error messages when it failed to find a course. I also replaced the original selection sort with a more efficient and complex merge sort algorithm, this improved scalability and runtime performance of the application. I also cleaned up input validation, cleared up the out formatting and implemented safeguards to prevent operations on the data before any data is loaded. I included a more robust search algorithm which is now case-insensitive for a more user-friendly application. With this enhancement I met several course outcomes that I planned to meet in module one.

## Reflection on the Enhancement Process
During the enhancement of this artifact, I was reminded of the importance of selecting appropriate algorithms and understanding their impact on performance and correctness. I also learned that even small logical errors can significantly alter the functionality of a program. Transitioning from a selection sort to a merge sort also deepened my understanding of algorithm efficiency and tradeoffs. For example, the balance between time complexity and memory usage. One challenge I faced while enhancing this artifact was debugging my search function. The search function was returning results, and sometimes it even returned the correct result. This meant that issue was not immediately obvious and required careful analysis of how string comparisons worked and how loop conditions impact program execution. Another challenge was implementing the merge sort algorithm correctly, as it required managing multiple subarrays and using proper merge logic.
