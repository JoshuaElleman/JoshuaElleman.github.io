# Enhancement Three
## Artifact Overview
Artifact three is a python CRUD module I used in CS 340 to connect to an animal shelter database for use in a dashboard. It was created a year ago in April of 2025. I included this artifact because I wanted to use the CRUD module to connect to a local database I had been maintaining this year that held information about the results of board game plays that I had with my family gaming group. 

## Justification for Inclusion in ePortfolio
 The enhanced artifact demonstrates a full-stack application that integrates frontend development, backend data management, and data visualization. This project highlights my ability to design and implement a solution to a real-world problem by analyzing board game outcomes. The use of Dash demonstrates my ability to build interactive user interfaces driven by callback-based logic. The integration of MongoDB through the python CRUD module highlights my understanding of database design and abstraction. The inclusion of dynamic filtering and data aggregation for visualizations using Plotly shows my ability to manipulate data structures and present analytical insights effectively. The artifact was significantly improved from its initial version. In the initial version all the data manipulation and the dashboard for the animal shelter data was done through a Jupyter Notebook. In the enhancement I have removed the need for a notebook and used Dash and Plotly directly to build a standalone local application. I also used Docker to build a container for the application so that my end users could run the application and ensure they have all the necessary libraries and technologies to run the application within the container.

## Reflection on the Enhancement Process
Reflecting on the process of enhancing this artifact I learned the importance of aligning data structures across all layers of an application. One of the challenges I faced was ensuring consistency between how data is stored in MongoDB and how it is processed and displayed in the dashboard. Issues such as type mismatches and filtering logic required careful debugging and reinforced the importance of data normalization. I also gained a deeper understanding of event-driven programming through Dash callbacks, specifically in managing state and ensuring components update correctly in response to user interactions.
